◆ IoTPApplication

typedef void* IoTPApplication

A handle representing an IBM Watson IoT Platform MQTT application client. A valid client handle is available following a successful call to IoTPApplication_create()

Function Documentation

◆ IoTPApplication_create()

DLLExport IOTPRC IoTPApplication_create ( IoTPApplication **  application,
IoTPConfig *  config 
)

IoTPApplication_create: Creates IBM Watson IoT platform application client.

This function creates IBM Watson IoT Platform Application client. The client handle is initialized with IoTPConfig object.

Parameters
application- A pointer to IoTPApplication handle.
config- A pointer to IoTPConfig handle.
Returns
IOTPRC - Returns one of the following codes:
  • IOTPRC_SUCCESS for success
  • IOTPRC_INVALID_HANDLE if handle is not valid
  • IOTPRC_INVALID_NOMEM if system runs out of memory

Use IoTPApplication_destroy() API, to destroy a client initialized using this API.

IoTPApplication_sendEvent: Publishs events for the application to the IBM Watson IoT service, or on behalf of other devices.

Parameters
application- A valid application handle
typeId- Device type ID
deviceId- Device ID
eventId- Event id to be published e.g status, gps
data- Payload of the event
formatString- Format of the event e.g json
qos- QoS for the publish event. Supported values : QoS0, QoS1, QoS2
props- MQTT V5 properties
Returns
IOTPRC - Returns one of the following codes:
  • IOTPRC_SUCCESS for success
  • IOTPRC_INVALID_HANDLE - if handle in invalid
